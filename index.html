Data on the Web

  • Data makes sense in the context of its retrieval
    • { id: "alice.com#me", knows: "bob.com/#me" }
      { id: "bob.com#me", name: "Bob" } .
  • APIs carry the context of the data in their documentation

Resource Description Framework (RDF)

  • A method for describing and exchanging graph data.

<something> <has a relation to> <something else> .

Linked Data on the Web

  • Linked Data links different pieces of information together using URIs:
    • alice.com/#me foaf:knows bob.com/#me .
      bob.com/#me foaf:name "Bob" .
  • These identifiers create interlinking between data
    and allow for the dereferencing of links
    • The http://xmlns.com/foaf/0.1/name property
      describes the name of a Person.

JSON and JSON-LD.

  • JavaScript Object Notation (JSON) is a lightweight, text-based, language-independent data interchange format.
  • JSON-LD is a JSON-based format to serialize Linked Data, designed to integrate into systems that already use JSON.
  • JSON-LD does not provide any guarantees about its form.

The JSON-LD context

  • Added using the @context key
  • Provides the context of the json object
    • It maps the terms of the JSON object to IRIs.
  • Can describe complex mappings
    • mapping, typed values, collection ordering, ...
  • see the spec for more information

When can you process JSON-LD as JSON?

  • If you can guarantee that no actor in the chain processes it as RDF.

    • RDF makes no guarantees on the form of a JSON-LD document.

    • As long as everybody in the chain treats it as JSON, you can process it as such.

When can you process JSON-LD as JSON?

  • By giving it a specific Frame
    using JSON-LD Framing

    • By using JSON-LD Framing,
      we can enforce a specific layout upon the data.

    • This way, we do not need to worry about
      the format of the RDF in the intermediate steps.

When can you process JSON-LD as JSON?

  • We can process the JSON-LD document as RDF.

    • We can create any intermediary format
      JSON, XML, ... directly from the RDF.

    • For this, there are mulitple approaches:
      querying the data using SPARQL,
      processing triples directly, ...

JSON-LD Framing

JSON-LD Framing allows developers to force a specific tree layout to a JSON-LD document.

You can find the spec HERE

JSON-LD Framing

  • Framing works by matching on a given structure
    of identifiers, types, properties, ...
  • It provides flags for specific functionality

    • the @requireall flag requires all properties in the frame object to be matched.

    • the @explicit flag will only output properties explicitly declared in the frame.

JSON-LD Framing

  • We can enforce different layouts
  • ... for the same data.

Processing the data as RDF

  • We can use multiple approaches
    • Processing the triples from the document indvidually
    • Using a querying approach such as SPARQL
    • ...
  • If interested in this approach, please contact us for more information!

JSON verification

  • JSON Schema is a popular approach.
  • Note that this does not verify whether the contents of documents in the @context change
  • There are no strong guarantees for the resulting RDF graph

JSON-LD / RDF verification

  • Validation of RDF graphs make use of shapes
  • Popular options are
    • SHACL
    • SHEX
  • Example usage can be found in the SHACL playground.

Tooling

  • There is tooling for JSON-LD in multiple languages:
    https://json-ld.org/#developers
  • For Javascript / Typescript:

    • jsonld.js

    • jsonld-streaming-parser.js
      jsonld-streaming-serializer.js

jsonld.js

  • Maintained by DigitalBazaar
    • Commercial support for this library is available upon request from Digital Bazaar
  • General purpose library for working with JSON-LD
  • Supports all functionality:
    • parsing, serializing, framing, compacting, expanding, ...
  • Supports JSON-LD 1.1

jsonld-streaming-parser.js /
jsonld-streaming-serializer.js

  • Maintained by the Comunica Association
    • Support is available through the Comunica Association
  • Supports JSON-LD 1.1 Supports parsing & serializing of JSON-LD using streams.
  • Supports JSON-LD 1.1

General RDF tooling

  • There is a lot of tooling to work with RDF in general in the JS / TS ecosystem
  • See the RDF-JS specification
  • And a curated list of complying libraries
    • Many of these also fall under the Comunica Association,
      or are actively maintained by other parties.
  • Feel free to contact us for more information!

Verification tooling

  • SHACL is on W3C recommendation track
  • Available libraries in JS/TS

    • TopBraid SHACL API by TopQuadrant

    • rdf-validate-shacl by zazuko
  • TopQuadrant used to provide a SHACL library for JS/TS, but now forwards this to the zazuko implementation. I have no knowledge of the level of enterprise support for the libraries sadly.
